# Chaos Proxy

HTTP-прокси с эмуляцией сетевых сбоев.

## 📖 Описание

Chaos Proxy — это HTTP-прокси, который позволяет искусственно создавать проблемы сети:
- Задерживать ответы на заданное время
- Возвращать ошибки вместо реальных ответов
- Обрезать тело ответа

## 🛠 Технологии

- **C++17** — современный стандарт C++
- **POSIX Sockets / Winsock** — работа с сетью
- **std::thread** — многопоточная обработка клиентов
- **std::mutex** — синхронизация вывода логов
- **CMake** — система сборки

**Никаких внешних библиотек!** Только стандартная библиотека C++17 и системные вызовы.

## 📝 Использование

### Базовый запрос

Прокси перенаправляет запросы на целевой сервер:

```bash
curl "http://localhost:8080/httpbin.org/get"
```

### Эмуляция задержки

Добавьте параметр `delay` (в миллисекундах):

```bash
# Задержка 2 секунды перед ответом
curl "http://localhost:8080/httpbin.org/get?delay=2000"
```

**Что происходит:** Прокси получает запрос, ждет 2 секунды, затем соединяется с httpbin.org и возвращает ответ.

### Эмуляция ошибок

Добавьте параметр `error_rate` (вероятность от 0.0 до 1.0):

```bash
# 100% шанс получить ошибку 503
curl "http://localhost:8080/httpbin.org/get?error_rate=1.0"

# 50% шанс получить ошибку
curl "http://localhost:8080/httpbin.org/get?error_rate=0.5"
```

**Что происходит:** Если срабатывает ошибка, прокси сразу возвращает 503 Service Unavailable, не обращаясь к целевому серверу.

### Обрезка ответа

Добавьте параметр `truncate=true`:

```bash
# Тело ответа будет обрезано на 50%
curl "http://localhost:8080/httpbin.org/html?truncate=true"
```

**Что происходит:** Прокси получает полный ответ от сервера, обрезает тело наполовину и отправляет клиенту.

### Комбинирование параметров

Можно использовать несколько параметров одновременно:

```bash
# Задержка 1 секунда + 30% шанс ошибки
curl "http://localhost:8080/httpbin.org/get?delay=1000&error_rate=0.3"

# Задержка + обрезка
curl "http://localhost:8080/httpbin.org/html?delay=500&truncate=true"
```

## 🔍 Как это работает

### Архитектура

1. **TcpServer** создает серверный сокет и слушает порт
2. При новом подключении создается отдельный `std::thread`
3. **ClientHandler** в потоке:
   - Читает HTTP-запрос от клиента
   - Парсит запрос через **HttpParser**
   - Извлекает query-параметры
   - Применяет задержку через **ChaosEngine**
   - Проверяет необходимость ошибки
   - Соединяется с целевым сервером
   - Пересылает запрос
   - Получает ответ
   - Применяет обрезку тела
   - Отправляет ответ клиенту

### Пример прохождения запроса

```
Клиент -> Прокси (?delay=1000) -> httpbin.org

1. curl отправляет GET /get?delay=1000 на localhost:8080
2. Прокси принимает соединение, создает поток
3. ClientHandler читает запрос
4. parseRequest извлекает: path=/get, query_params={delay: 1000}
5. applyDelay ждет 1000 мс
6. shouldInjectError проверяет error_rate (нет -> false)
7. connectToHost соединяется с httpbin.org:80
8. send отправляет GET /get на httpbin.org
9. recv получает ответ от httpbin.org
10. truncateBody проверяет truncate (нет -> оставляем как есть)
11. send отправляет ответ клиенту
12. close закрывает соединения
```

- Работает только с HTTP (не HTTPS)
- Не поддерживает chunked transfer encoding
